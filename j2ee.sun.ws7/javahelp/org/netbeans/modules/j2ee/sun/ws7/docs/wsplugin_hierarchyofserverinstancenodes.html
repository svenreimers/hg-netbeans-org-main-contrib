Hierarchy of Sun Java System Web Server Instance Nodes

This topic explains the actions you can invoke from the right-click pop-up menu that you access from the Sun Java System Web Server 7.0 node and its subnodes for local server instances. When you expand the Sun Java System Web Server 7.0 node, the configured server instance tree displays.

Action Description
Sun Java System Web Server 7.0 Node
Start/Stop Server Opens the Server Status dialog box where you can start and stop the local server.
Remove Lets you remove the server registration from the IDE.
Show Admin Console Launches the administration console for the server.
Sun Java System Web Server 6.1 > server-instance Node
Start Instance/Stop Instance Lets you start and stop the local server instance.
View Server Logs Displays server logs.
View Access Logs Displays access logs.
  • You cannot start or stop a remote server instance.

Virtual Servers

Each server instance node has one or more virtual servers defined under its node. Web applications are deployed on a virtual server.

Expand the server instance node to see the virtual servers configured for each server instance. One of the virtual servers will always be defined as the default server (Default). Whenever you deploy a web application, it will be deployed on this default virtual server. You can define another virtual server as the default server by right-clicking the virtual server node and choosing Set as Default from the pop-up menu.

Each virtual server node has two subnodes:

  • Deployed Components. Expand this node to see all the web applications deployed on the selected virtual server. Using the contextual menu on a web application node, you can perform the following actions:
    • Undeploy the web application.
    • Disable the web application.
    • Display the properties of the web application.
  • Resources. Expand this node to see all the JDBC resources configured for the selected server instance. Using the contextual menu on a JDBC resource node, you can perform the following actions:
    • Delete the resource.
    • Display the properties of the JDBC resource.

Final Hierarchy

The final hierarchy of a Sun Java System Web Server 7.0 node may look like the one shown below.

Sun Java System Web Server 7.0
  --- instance1
      --- virtual-server1:80[Default]
          --- Deployed Components
              --- app1
              --- app2
          --- Resources
              --- jdbc/res1
      --- virtual-server2:81
  --- instance2
      --- virtual-server1:90
      --- virtual-server2:91
