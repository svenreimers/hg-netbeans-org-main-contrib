Using VCS with IDE Projects

See Also

The IDE's VCS support enables developer groups to synchronize project-specific metadata as well as the source files that make up their applications. Using VCS, you can place project metadata, such as project properties files and Ant build scripts, under version control.

  • The physical location of each project in relationship to the location of projects and libraries on its classpath must be exactly the same on every machine the project is checked out to. If any references are broken, the IDE displays a warning when you open the project. To avoid broken references, it is advisable to ensure that the location of projects in the VCS repository matches their location when they are checked out to a local working directory.

IDE Project Metadata Files

Depending on project type, the IDE generates different metadata files and stores them in the nbproject directory. The types of project metadata files in the IDE include:

  • nbproject directory
    • project.xml
    • build-impl.xml (standard projects only)
    • genfiles.properties (standard projects only)
    • project.properties (standard projects only)
    • private directory (standard projects only)
  • build.xml
  • You should not place your private directory into the VCS repository, as it contains information that is specific to your system's IDE installation. The IDE automatically ignores the private directory when running VCS commands. In addition, you should not put your dist or build directories into the repository.

To place IDE project metadata under version control:

  1. Register the directory that contains the project as a VCS working directory. For more information, see Setting Up VCS.
  2. In the Projects window, right-click your project's node and choose Clean Project to delete the build and dist directories.
  3. Add the project metadata to the VCS repository. If you are using CVS, for example, open the Files window (Ctrl-2), right-click the application's topmost node, and choose CVS > Add.
  4. In the Add dialog box, select Add All Local Files in Folder Contents and select Add the Folder Contents Recursively. Click OK.

    Note: the IDE automatically ignores the private directory and any files that are already in the CVS repository.

  5. Commit the project metadata to the VCS repository. If you are using CVS, right-click the topmost application node in the Files window and choose CVS > Commit.
  6. Enter a message describing your changes to the repository in the Commit dialog and click OK.

    The IDE uploads the new files to the repository and the VCS Ouput window opens to display the progress.
