CVS Command Reference

See Also

The following table describes the commands available when using the IDE's built-in or other command line CVS client with the included profile.

Main Menu Commands

CVS Command Description
Init Local Repository Initializes a CVS repository. Initialization is only necessary after creation of a new directory within which to store the repository.
Check Out Checks out a CVS modules from the CVS repository into your working directory.
Import Copies the whole working directory structure to the CVS repository. After using this command, delete the original sources in your system and check out the imported directory structure from the repository.

Contextual Menu Commands

CVS Command Description
Refresh Refreshes the status of the files in the selected folder.
Refresh Recursively Refreshes the status of all files in the selected folder, including those in all of the folder's subdirectories.
Commit Incorporates your changes to the CVS repository. When you choose this command, a dialog box prompts you to enter a log message describing the changes. When a template is present, it is added to the reason text area.
Update Updates the local version of the file with changes that other developers have committed to the CVS repository and, if necessary, merges local changes.
Add Adds local files to the CVS repository. This command is only available on files and directories whose CVS status is currently local. Following use of the Add command, the Commit command must be run in order to actually incorporate the file into the CVS repository.
Remove Removes the file from the CVS repository. The file is deleted locally and is scheduled for removal from the repository. The file is not removed from the repository until you run the Commit command.
Status Displays a window with the file's status information.
History > Log Displays a window with a full list of the file's revisions, tags, and commit history. You can also view revision history for a file in the Versioning window by right-clicking the file and choosing Show Versions.
History > History Displays information about the use of the checkout, commit, rtag, update, and release commands on the files in your working directory (command-line only).
History > Annotate Displays the file in a window where each line is annotated noting the revision in which it was last changed and who changed it.
Check Out Checks out a CVS modules from the CVS repository into your working directory.
Export Exports local copies of version-controlled files to a designated directory without the CVS administrative directories.
Editing > Edit Enables the local copy of a file to be editable. Also sends notification to users who have requested edit notification for this file (command-line only).
Editing > Undo Edit Abandons changes to the working file and reverts to the repository version on which the file is based (command-line only).
Editing > Editors Lists the users currently working on the file (command-line only).
Watches > Watch On Enables developer notification of the need to run the Edit command prior to editing files (command-line only). By default, the IDE calls the Edit command automatically when you start editing a watched file.
Watches > Watch Off Disables developer notification about work on files. Working files created by CVS are given read/write attributes (command-line only).
Watches > Set Watch Displays a list of available watch actions and adds the current user to the list of people to receive notification of work done on the files (command-line only).
Watches > Watchers Lists the users currently watching changes to files (command-line only).
Locking > Lock Locks the file so that only you can edit it (command-line only).
Locking > Unlock Unlocks the file to allow other users to edit it (command-line only).
Branching and Tagging > Remove Tag Removes a symbolic tag from the repository. This also updates your file with the HEAD revision.
Branching and Tagging > Add Tag Adds a symbolic tag or a branch tag to the file. The desired revision number and tag type can be specified in the dialog box.
Branching and Tagging > View Branches Displays a graphical hierarchy of all the revisions and branches for a file. Click Diff to view differences between any two revisions.
Diff Graphical Displays the differences between your working file and the repository version on which it is based. You can also specify two file versions to compare. The two versions are displayed next to each other with the differences highlighted.
Diff Textual Displays the differences between your working file and the repository version on which it is based. You can also specify two file versions to compare. The differences between the two versions are output as text.
Patch Genereates a patch from the differences of two or more sources (command-line only).
Apply Patch Applies a patch from the differences of two or more sources (command-line only).
List Modules Provides a list of modules available in the CVS repository.
Login Logs you in to the CVS repository.
Logout Logs you out of the CVS repository.
