Configuring Version Control Settings

See Also

The IDE allows you to edit both global VCS settings and settings specific to a local VCS working directory. The Versioning customizer allows you to edit VCS settings and other advanced options for a VCS working directory. You can also configure global VCS settings in the IDE's Options window.

To edit settings for a local VCS working directory:

  1. Choose Versioning > Versioning Manager.
  2. In the Versioning Manager, open the VCS working directory's customizer by selecting it's node and choosing Edit.
  3. In the Profile Tab, select the profile for the VCS program you are using from the list of those currently installed in the Version Control System Profile combo box.

    Note: the remaining settings differ depending on which profile is currently selected.

  4. Edit any settings you wish to change.
  5. (Optional) In the Advanced Tab, select any advanced options you require.
  6. (Optional) In the Environment Tab, disable any system environment variables you don't want to use with the IDE. You can also create and edit user-defined environment variables.

    Note: Most environment variables must be set outside of the IDE.

  7. (Optional) In the Properties Tab, configure any of the version control system's environment variables you want to use with the IDE.
  8. Click OK to close the customizer.

    The IDE applies the settings to the designated local working directory.

  • Changes made in the customizer apply only to the local working directory for which they are set.

To edit global VCS settings:

  1. Choose Tools > Options from the main window.
  2. In the Options window, expand the Source Creation and Management node and select the Version Control Settings node.
  3. Enter the settings for the options you require.

    You can configure the following options:

    • Advanced Mode Notification. If true, enables prompting for advanced options when invoking VCS commands.
    • Auto Refresh. Sets the frequency with which versioned directories automatically refresh file status information. It is recommended that you set this property to For Newly Opened Folders because automatic refreshes on large directories can take a long time.
    • File Status Annotation Display. Sets the whether or not files' status annotations are displayed and at what level of detail.
    • Home Directory. Sets the location corresponding to the system's HOME directory. This directory is used as the location for configuration files, such as the .cvspass file for CVS filesystems.
    • Offline Mode. If True, disables automatic refreshing of status information. File status information is still available with the Refresh command.
    • Override Settings of Versioned Directories. If True, ignores options set for specific working directories. Changes are applied to new versioned directories only, existing working directories are not affected.
  4. If you need to edit VCS commands for a profile, expand Version Control Settings > Generic VCS Profiles, select the profile's node, and click the ellipsis (...) for the appropriate property.

    The IDE applies the settings to the designated VCS profile.

  • You can also configure how a VCS working directory's versioning information is displayed by right-clicking its node () in the Versioning window and choosing Properties.
