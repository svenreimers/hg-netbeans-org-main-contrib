Using VCS Groups

See Also

VCS groups simplify the process of working with version-controlled files by organizing related files together. Once organized into groups, you can use the VCS Groups window to track changed files, prepare groups of files for check-in, and perform VCS commands on the entire group.

To open the VCS Groups window, choose Window > Versioning > VCS Groups from the main window.

Adding Files to VCS Groups

You can add files to a group manually using the Include in VCS Group command or have them added automatically whenever they are altered.

To add files to the Default Group automatically:

  1. Right-click the top-level VCS Groups node and choose Properties.
  2. In the VCS Groups property sheet, select Move Modified Files to Default Group in the Auto Addition combo box.
Files are automatically added to the Default Group whenever they are changed. If necessary, you can then move the files to another group.

Preparing Grouped Files for Check-in to the Repository

Prior to committing a group of files to the repository, you can use the Verify command to insure that each of the files exist in the repository, are up to date (CVS only), and have been changed.

To verify a group of files for check-in: resolve any problems with your grouped files

  1. Right-click the node of the group containing the files you want to verify and choose Verify. The Group Verification dialog box opens, displaying all files found to have problems sorted under one of four tabs:
    • Do Add
    • Do Update
    • Not Locked
    • Not Changed
  2. In each tab, select the check box to activate the radio buttons' functions if necessary.
  3. Select the files you wish to include in each tab's file list and select the appropriate radio buttons.
  4. After configuring settings for each tab, click Correct Group.

    The IDE carries out the specified commands and displays a status report in the VCS Ouput window.

  • To resolve all of the problems with grouped files simultaneously, set all of the options on all of the tabs in the Group Verification dialog box before you click Correct Group.

Adding and Removing VCS Groups

By default, the VCS Groups window contains the Default group alone, however, you can also create additional groups within which to organize your versioned files.

To create a VCS group:

  1. Right-clicking the top-level VCS Groups node and choose Add VCS Group.
  2. Enter the desired name for the group and click OK.

    The IDE the list of available groups in the VCS Groups window.
