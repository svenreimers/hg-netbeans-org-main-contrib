Portable Object Adapter

The IDE provides support for Portable Object Adapters (POAs), which enable you to construct object implementations that are portable between different ORBs. The POA node is visible as the root of each CORBA server's POA hierarchy within the Explorer window.

The POA hierarchy within the Explorer represents a hierarchy of POAs within a CORBA server and is formed by the following nodes:

  • POA
  • Servant
  • ServantManager
  • Default Servant
  • Specifying New Child POAs and Removing POAs
  • Registering a New POA Activator
  • Registering Servants
  • Unregistering POA Hierarchy Elements
  • Displaying and Changing POA Properties
