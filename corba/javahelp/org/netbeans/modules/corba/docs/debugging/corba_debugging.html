Debugging CORBA Programs

You can use the IDE's debugger with your CORBA client and server programs. There are few things you must consider:
  • You must debug client and server programs separately; you cannot start the debugger on the client side and trace the remote method call into the server. ( What you should do is start the debugger on the server side and then run the client; the debugger will show the server executing in response to the request.)
  • You cannot use the Forte for Java debugger for processes that are started outside of the IDE. In particular,  you cannot debug CORBA server processes that are started from an implementation repository.
  • If you want to debug a server-side program that is already running, you need to unbind it from the naming service before restarting it. (If you don't, you will get an exception when the binding code executes.)
    • If you have the naming service attached in the Naming Service Browser (on the Runtime tab of Explorer), right click on the node for the server, choose Unbind from the context menu.
    • If it is not attached, you will need too kill and restart tnameserv:

      • Either using shell utilities (ps and kill) or your operating system's task manager, kill tnameserv's process.
      Restart it using the same arguments as before (tnameserv -ORBInitialPort 1050).
  • If you only want to debug the client side, it is not necessary to restart the naming service and rebind the implementation instance.

Setting Breakpoints

Use the standard IDE procedures for setting breakpoints. For example, to set a breakpoint in the Hello World example's implementation class:
  1. Go to the HelloImpl class in the editor. Locate the line containing the for() loop in the code which you entered earlier.
  2. Place the text caret on this line, and press CTRL-F8 to set a breakpoint on this line.
  3. Select the HelloServer class in the Explorer, and press F5 to start the server in the debugger.
Execute the client as you did before. Server-side execution will pause at your breakpoint. Just as when debugging other Java objects, you can freely make use of tracing, watch variables, and other debugging techniques.
Note: If you are running the IDE on HotSpot you must disable HotSpot before restarting the client.
