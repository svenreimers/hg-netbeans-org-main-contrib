Specifying the ORB You Are Using

For the CORBA module to generate stub and skeleton files that will work with a particular ORB product, you need to set properties identify that ORB product:
  1. Open the Project Settings dialog by choosing Project > Settings from the main menu.
  2. Locate the CORBA Settings node and click on it. The CORBA settings appear on the right half of the dialog.
  3. Make sure that the ORB Version property is set to the ORB you will be using.
  4. Leave the Project Settings window open and click on the CORBA Settings node to display its sub-nodes.
  5. Click on the sub-node for the ORB you are using; the properties for that ORB appear on the right half of the dialog.
  6. When you develop CORBA programs, the IDL compiler generates "stub," "skeleton," and other supporting files and puts them in the package with the file you actually write. You do not reference these files directly in code you write, so by default they are hidden. If you want to see them, set the Hide Generated Files property to false.
  7. Make sure the other properties, such as the Generated Skeletons property, are set to values that are appropriate for your project.
  8. If you are going to specify the external IDL compiler executable as a property:
    • Click on the Expert tab.
    • Click on External IDL Compiler, then click on the "..." symbol.
    • Type in full path to the IDL compiler.
Note that these settings apply to the current development project. Different projects can be configured differently.
