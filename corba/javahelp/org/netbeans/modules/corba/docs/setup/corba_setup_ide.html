Specifying an ORB

For the CORBA module to generate stub and skeleton files that will work with a particular ORB product, you need to set properties that identify that ORB product.

To examine and modify CORBA property settings:

  1. Open the Project Settings dialog box by choosing Project Settings.
  2. Locate the CORBA Settings node and click it. The CORBA settings appear on the right half of the dialog box.
  3. Make sure that the ORB Version property is set to the ORB you will be using.
  4. Leave the Project Settings window open, and expand the CORBA Settings node to display its sub-nodes.
  5. Click the sub-node for the ORB you are using; the properties for that ORB appear in the right half of the dialog box.

    When you develop CORBA programs, the IDL compiler generates stub, skeleton, and other supporting files and puts them in the package with the file you actually write. You do not reference these files directly in the code you write, so by default they are hidden. If you want to see them, set the Hide Generated Files property to false.

  6. Make sure the other properties, such as the Generated Skeletons property, are set to values that are appropriate for your project.

To specify the external IDL compiler executable as a property:

  1. Click the Expert tab for the selected ORB.
  2. Click External IDL Compiler, then click the ellipsis ("...") symbol.
  3. Type in the full path to the IDL compiler.

These settings apply to the current development project. Different projects can be configured differently.
