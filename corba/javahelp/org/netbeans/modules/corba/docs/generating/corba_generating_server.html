Generating Server-side Files With the CORBA Wizard

You can generate server-side files with either the CORBA wizard or the CORBA templates. Before generating any code:

  • Make sure the IDE is set up for the ORB product you want your CORBA programs to work with.
  • Have an existing IDL file to use the CORBA wizard.
  • If you are using the naming service, make sure that one is running.

    To create a server file using the CORBA Wizard:

    1. Choose Tools CORBA Wizard to open the CORBA Wizard.
    2. Select the IDL file on which your client program will be based and click Next.
    3. Indicate which files the wizard should generate, then click Next to proceed:
      • If you are writing a server based on existing IDL interface, check the Create Server and Create Implementation boxes.
      • If you are writing both the IDL interface and the server program, check the Create Server and Create Implementation boxes.
      • If you are writing the IDL interface, and a server program that uses it, and a client program, check the Create Server, Create Implementation, and Create Client boxes.
    4. Choose an ORB implementation version and a binding method. Click Next to proceed.
    5. Select a root interface if you want one other than the preselected one shown. Click Next to proceed.
    6. If you chose to use the Naming Service as your binding method, provide information about the binding method:
      1. Select CORBA Naming Service.
      2. Click Bind Context.
      3. In the CORBA Panel dialog box, enter the name, kind of the Naming Service and provide either the URL or IOR string for the naming service and click OK. For information on obtaining this information, see Starting the Naming Service Browser.
      4. Enter the name of the naming service you are using (for example, JDK).
      5. Enter the kind of service (for example, Naming).
      6. Click Next.
    7. Click Finish to begin code generation.

      The CORBA Module generates a local copy of the IDL you specified and Java source files named <IDL_interface_name>Server. For example, if you specified hello.idl, the wizard will create a local copy of hello.idl and two .java files named helloServer.java and helloImpl.java.
