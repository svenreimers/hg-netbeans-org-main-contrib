Generating Client-side Files

You can generate client-side files with either the CORBA wizard or the CORBA templates. Before generating any code, make sure the IDE is set up for the ORB product you want your CORBA programs to work with.

Using the CORBA Wizard

  1. From the main window, choose Tools  CORBA Wizard to open the first page of the CORBA Wizard.
  2. On this page::
    1. Type in the name of the IDL file on which your client program will be based. This can be either an existing file or a new file (which you create on page three of the wizard).
    2. Select the package in which you want the wizard to put the files it generates.
  3. On the second page of the wizard, indicate which files the wizard should generate:
    • If you are writing a client for an existing server, check only the Create Client box.
    • If you are writing a client for an existing server, your client will be based on an existing IDL file and you do not need to check the Create IDL box.
  4. On the third page of the wizard, choose the settings that describe the implementation that your client will be working with.

    5. If you are writing a client for an existing server, you need to know whih binding method the server uses and whether the server is tie-based or inheritance-based.
  5. On the fourth page of the wizard, identify the IDL on which your client program will be based.
    • If the IDL is in a file, click the Browse button and select the file.
    • If the IDL is in an interface repository, browse the CORBA Interface Repository node and select the correct interface. (To do this, you must have already started the CORBA vendor's interface repository and started up the CORBA Module's Interface Repository Browser.)
  6. On the fifth page of the wizard, review the description of the actions the wizard will take, and click the Finish button to begin code generation.

    7. If you are generating a client for an existing server, the description will read, "The CORBA Wizard will now import IDL, generate Client."
  7. The CORBA Module generates a local copy of the IDL you specified and a java source file named <IDL_interface_name>Client. For example, if you specified hello.idl, the wizard will create a local copy of hello.idl and helloClient.java.

Using the CORBA Templates

  1. From the main window, choose File  New to open the Template Chooser.
  2. From the CORBA template category, select the ClientMain template.
  3. Use the fields on the right side of the Template Chooser to specify both a name for your client file and the package in which it should be generated.
  4. Click the Finish button.

    5. The CORBA module creates a CORBA client source file, named <Name_You_Input>.java.
  5. In addition to generating client program source code with the template, you need to copy the IDL on which your client is based to the client program's package.
