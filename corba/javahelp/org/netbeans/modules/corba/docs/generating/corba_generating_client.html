Generating Client-side Files With the CORBA Wizard

You can generate client-side files with the CORBA template. You can also use the CORBA wizard in addition to add some basic information to expedite the process. Before generating any code, make sure the IDE is set up for the ORB product you want your CORBA programs to work with. You must have an existing IDL file to use the CORBA wizard.

To create a client file using the CORBA Wizard:

  1. Choose Tools CORBA Wizard.
  2. Select the IDL file on which your client program will be based and click Next.
  3. Indicate which files the wizard should generate, then click Next to proceed:
    • If you are writing a client for an existing server, check only the Create Client box. (Your client will be based on an existing IDL file.)
    • Choose the ORB implementation and binding method you want to use. Click Next to proceed.
    • Select the root interface you want and click to Next.
    • If you chose to use the Naming Service as your binding method, provide information about the binding method:
      1. Select CORBA Naming Service.
      2. Click Bind Content.
      3. In the CORBA Panel dialog box, enter the name, kind of the Naming Service and provide either the URL or IOR string for the naming service and click OK. For information on obtaining this information, see Starting the Naming Service Browser.
      4. Enter the name of the naming service you are using (for example, JDK).
      5. Enter the kind of service (for example, Naming).
      6. Enter either the URL for the naming service IOR or paste IOR string directly into the IOR text field.
    • Review the description of the actions the wizard will take, then click Finish to begin code generation.

      The CORBA module generates a local copy of the IDL you specified and a java source file named IDL_interface_nameClient. For example, if you specified hello.idl, the wizard creates a local copy of hello.idl and helloClient.java.
