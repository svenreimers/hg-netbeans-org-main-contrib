Using the NetBeans Integration of the Debugger

To launch the debugger on the main class of your main project, just click Omniscient Debug Main Project in the NetBeans toolbar or Run menu.

You may also right-click any project and select Omniscient Debug Project.

You may debug a single file using Run | Run File | Omniscient Debug File. (Also in the context menu for Java sources.) The file must either have a main method or be a JUnit test.

Once the debugger is running, NetBeans is not involved; refer to the debugger user manual for details on how to use it.

The initial configuration for the debugger is kept in nbproject/private/cache/.debuggerDefaults and there is a omnidebug.xml Ant script in the same directory which you may run by hand if you want to repeat the same debugger run. The IDE uses the debugger's default configuration except for the following:

  • The class name is of course set according to your main class or unit test.
  • The class path is whatever you configure for the runtime classpath of your project.
  • All classes in the project being debugged are instrumented. Additionally, all classes open in the editor are instrumented.
  • Source associations for all classes in the class path are made available to the debugger where possible.
  • If your main class imports some AWT or Swing classes, the IDE guesses that it is a GUI application and asks the debugger to not stop recording when the main method exits.
  • An attempt is made to pass the same system properties to your main class or test as the regular project's Ant script would use.
