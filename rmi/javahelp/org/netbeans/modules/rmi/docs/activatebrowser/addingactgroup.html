Adding or Customizing an Activation Group

Once you have created an instance of the Activation browser, you can add one or more activation groups using the Add Activation Group dialog box. You can also customize an activation group by modifying the properties of group through the Customizer dialog box.

To open the Add Activation Group dialog box:

  1. Right-click on an instance of the Activation browser in the Explorer's Runtime pane.
  2. Select Add Activation Group from the contextual menu.

To customize an Activation group:

  1. Right-click on an Activation group in the Explorer's Runtime pane.
  2. Select Customize from the contextual menu.
  3. Modify the information in the Customizer dialog box and click OK.

    For an explanation of the fields in the dialog box, see the steps for adding an activation group.

To add an activation group, supply the following information:

  1. Class Name. The name of the activation group class. Click on the ellipsis button to use a file chooser to select the class. A null group class name indicates the system's default Activation group implementation.
  2. Location. The path where the class is located. The path is automatically supplied when you select the class from the file chooser.
  3. Command Environment. The environment settings for the activation group. You can specify:
    • Process. The path to the JVM used by the activation group.
    • Item. (Optional) Any arguments you want to add when initializing the JVM.
  4. Properties. Properties that specify overrides to the JVM.
  5. Serialized Data. The object's initialization data contained in marshalled form. Click the ellipsis (...) button to add or customize an object. Click Create New in the Customize Bean dialog box to select an object. Once an object is selected, you can modify it in the Customize Bean dialog box if desired before clicking OK to add it to the Serialized Data field.
